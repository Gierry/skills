# 微信公众号文章完整生成

一站式完成从原始文案到可发布HTML的完整流程。

## 核心功能

将用户提供的原始文案，通过三个步骤自动处理，最终生成包含配图的微信公众号HTML文章。

## 完整流程

```
原始文案
    ↓
【步骤1】文案转写 (helleAI-wechat-step1-text-to-md)
    - 分析文案类型和受众
    - 生成3-5个爆款标题
    - 转写为结构化Markdown
    - 优化段落和重点标记
    ↓ 输出: {slug}.md
【步骤2】配图生成 (helleAI-wechat-step2-md-analyze-images)
    - 分析配图需求和位置
    - 生成中文图片提示词
    - 检查并选择可用的图片生成供应商
    - 调用图片生成API生成图片
    - 压缩图片优化体积
    - 上传到微信公众号获取URL
    ↓ 输出: image-mapping.json + 图片文件
【步骤3】HTML整合 (helleAI-wechat-step3-md-to-html-by-images)
    - 转换Markdown为HTML结构
    - 插入微信图片URL到对应位置
    - 应用微信公众号规范样式
    - 验证图片数量一致性
    ↓ 输出: {slug}-final.html
最终HTML (可直接发布到微信公众号)
```

## 使用方法

### 基础用法


直接提供文案，AI会自动执行三个步骤：

```
"请使用 helleAI-wechat-article-complete 处理这段文案：

[粘贴你的原始文案]
"
```

### 高级用法

指定特定要求：

```

- 目标受众：职场新人
- 风格：专业严谨
- 配图数量：3-4张
- 视觉风格：清新手绘插画

文案内容：
[粘贴文案]
"
```

### 步骤1: 文案转写

**调用技能**: `helleAI-wechat-step1-text-to-md`

**详细流程**:

1. **接收并分析原始文案**
   - 识别文案类型（教程/评测/观点/故事）
   - 提取核心观点和关键信息
   - 判断目标受众和情绪基调
   - 确定适合的写作风格

2. **生成爆款标题组**
   - 悬念类："99%的人都不知道的..."
   - 利益点类："3个方法让你..."
   - 情绪共鸣类："终于有人说出了..."
   - 数字类："5分钟学会..."
   - 对比类："从XX到XX，我..."

3. **创作结构化正文**
   - 使用Markdown标记（`##` 标题、`**加粗**`、`- 列表`）
   - 每段不超过3行
   - 重点内容加粗
   - 代码块只用于真正的代码
   - 重点提醒用引用块

4. **保存Markdown文件**
   - 文件命名：`wechat-articles/YYYY-MM-DD/{slug}/{slug}.md`
   - 包含frontmatter（标题、日期、slug）
   - 自动备份已存在的文件

5. **输出步骤1报告**
   - 显示生成的标题选项
   - 统计字数和章节数
   - 提供文件路径

**输出**:
- `{slug}.md` - Markdown文件
- 标题选项列表
- 文章统计信息


### 步骤2: 配图生成

**调用技能**: `helleAI-wechat-step2-md-analyze-images`

**详细流程**:

1. **读取并分析Markdown**
   - 识别文章结构（章节、小节）
   - 找出核心观点转折处
   - 确定需要视觉化的段落
   - 评估长文排版间隔需求

2. **生成配图分析JSON**
   - 确定配图位置（封面、章节图、结尾图）
   - 设定统一视觉风格
   - 编写中文图片提示词
   - 指定图片尺寸（宽900px）

3. **确认配图方案**
   - 使用AskUserQuestion展示方案
   - 显示图片数量和位置
   - 预估生成时间
   - 等待用户确认

3.5. **检查图片生成供应商配置**
   - 默认使用 Helle.ai
   - 如果 Helle.ai 未配置，查找其他已配置的供应商
   - 按优先级: Helle.ai → Google → OpenAI → DashScope → Replicate
   - 如果没有任何供应商配置 API Key，报错并终止
   - 显示选定的供应商和模型信息

4. **生成图片**
   - 调用 `helleAI-image-gen` 技能
   - 使用步骤3.5选定的供应商和模型
   - 顺序生成，每张报告进度

5. **压缩图片**
   - 调用 `helleAI-compress-image` 技能
   - 格式：WebP
   - 质量：80
   - 保留原图

6. **上传到微信公众号**
   - 获取access_token
   - 上传每张压缩图
   - 获取media_id和微信URL
   - 错误自动重试

7. **保存图片映射文件**
   - 创建 `image-mapping.json`
   - 记录每张图片的信息
   - 包含本地路径和微信URL
   - 统计压缩率

8. **输出步骤2报告**
   - 显示图片处理详情表格
   - 统计压缩效果
   - 提供文件位置

**输出**:
- `image-analysis.json` - 配图分析
- `image-mapping.json` - 图片映射
- `images/` - 所有图片文件
- 压缩统计报告


### 步骤3: HTML整合

**调用技能**: `helleAI-wechat-step3-md-to-html-by-images`

**详细流程**:

1. **读取输入文件**
   - 验证Markdown文件存在
   - 验证image-mapping.json存在
   - 如果缺失，提示错误并停止

2. **解析图片映射关系**
   - 读取image-mapping.json
   - 提取图片序号、位置、URL
   - 构建位置映射表

3. **转换Markdown为HTML**
   - `##` → `<h2>` 标题
   - `**加粗**` → `<span style="font-weight: bold;">`
   - `- 列表` → `<p>` + 蓝色圆点
   - ` ```bash ` → macOS窗口风格代码块
   - `> ⚠️ **重点**` → 重点贴纸样式

4. **插入微信图片URL**
   - 匹配位置描述（封面、第N章结束处）
   - 在对应位置插入图片HTML
   - 使用居中样式
   - 保持图片顺序

5. **验证图片数量**
   - 统计输入图片数量
   - 统计输出HTML中的图片数量
   - 验证两者完全相等
   - 如果不一致，报错并停止

6. **保存最终HTML**
   - 文件命名：`{slug}-final.html`
   - 备份已存在的文件
   - 不包含 `<html>`, `<head>`, `<body>` 标签
   - 从 `<section>` 开始

7. **输出最终报告**
   - 显示内容统计
   - 显示图片详情表格
   - 显示HTML规范检查结果
   - 提供下一步操作建议

**输出**:
- `{slug}-final.html` - 最终HTML文件
- 图片验证报告
- HTML规范检查结果

## 核心规则

### ⚠️ 最重要：图片数量必须完全一致

这是整个流程中最关键的规则：

- ✅ 步骤2生成N张图片 → 步骤3输出必须有N张图片
- ✅ 图片URL必须保持不变（使用微信公众号URL）
- ✅ 图片顺序必须保持不变
- ❌ 绝对禁止删除任何一张图片
- ❌ 绝对禁止添加原本不存在的图片
- ❌ 绝对禁止修改图片的URL

**原因**: 很多文章是操作流程教程，图片展示的是步骤截图。一旦少了任何一张图片，整个流程就会断裂。

### 微信HTML规范

- ❌ 禁止使用 `<div>` 标签（使用 `<section>` 替代）
- ❌ 禁止使用 `<ul><li>` 结构（使用 `<p>` + 样式化圆点）
- ✅ 所有样式必须内联
- ✅ 添加 `visibility: visible` 属性

### 内容质量标准

- **语言风格**: 口语化、亲切、易懂
- **段落控制**: 每段不超过3行
- **重点突出**: 核心内容用 `**加粗**`
- **信息完整**: 保留所有关键信息点
- **结构清晰**: 章节顺序、图文对应关系不变

## 常见问题

### Q1: 如果某个步骤失败怎么办？

**A**: 每个步骤都有备份和重试机制：

- **步骤1失败**: 重新执行，会自动备份旧文件
- **步骤2失败**: 
  - 图片生成失败：自动重试一次
  - 上传失败：提供手动上传指引
- **步骤3失败**: 检查文件是否存在，重新执行

### Q2: 可以只执行某个步骤吗？

**A**: 可以！直接调用对应的子技能：

```
只执行步骤1: "使用 helleAI-wechat-step1-text-to-md 转写文案"
只执行步骤2: "使用 helleAI-wechat-step2-md-analyze-images 配图"
只执行步骤3: "使用 helleAI-wechat-step3-md-to-html-by-images 生成HTML"
```

### Q3: 如何修改已生成的内容？

**A**: 
- **修改文案**: 编辑 `{slug}.md`，然后重新执行步骤3
- **修改配图**: 编辑 `image-analysis.json` 的提示词，重新执行步骤2
- **修改HTML**: 直接编辑 `{slug}-final.html`

### Q4: 生成的图片不满意怎么办？

**A**: 有两种方式：

1. **调整提示词**: 编辑 `image-analysis.json`，修改"中文提示词"字段，重新执行步骤2
2. **手动替换**: 直接替换 `images/` 中的图片文件，然后重新上传

### Q5: 预计需要多长时间？

**A**: 
- 步骤1: 1-2分钟（文案转写）
- 步骤2: 5-15分钟（取决于图片数量，每张约30秒）
- 步骤3: 1-2分钟（HTML生成）
- **总计**: 约10-20分钟

## 最佳实践

3. **数量适中**: 建议2-5张，不要过多

1. **预览效果**: 在微信编辑器中预览
2. **移动端测试**: 确认手机端显示正常
3. **图片加载**: 确认所有图片正常显示

## 注意事项

### ✅ 必须做到

1. **完整执行**: 三个步骤必须全部完成
2. **保留文件**: 不要删除中间生成的文件
3. **验证图片**: 确保图片数量一致
4. **检查规范**: 确保HTML符合微信规范

### ❌ 禁止行为

1. **不要跳过步骤**: 每个步骤都有其作用
2. **不要手动修改中间文件**: 除非明确知道影响
3. **不要删除图片**: 会导致步骤3失败
4. **不要修改文件结构**: 保持标准的目录结构

## 技术支持

如遇问题，请检查：

1. **环境变量**: 是否正确配置API密钥
2. **依赖技能**: 是否已安装所有必需技能
3. **网络连接**: 是否能访问API服务
4. **API配额**: 是否还有剩余配额
